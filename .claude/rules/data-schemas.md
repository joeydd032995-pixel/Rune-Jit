# Data Schema Rules

## Schema Validation Required

All JSON data files in `data/` must have a corresponding JSON Schema. Schemas live in `data/schemas/`:

```
data/
├── schemas/
│   ├── item-definition.schema.json
│   ├── npc-definition.schema.json
│   ├── shop.schema.json
│   └── skill-data.schema.json
├── skills/
│   ├── woodcutting.yaml      # validated against skill-data.schema.json
│   └── mining.yaml
└── shops/
    └── shops.json            # validated against shop.schema.json
```

## No Raw Jagex Binary Assets

Files in `data/` must be:
- JSON, YAML, CSV, or text
- Derived/computed data (not raw cache extracts)
- Properly licensed (open-source or community-documented)

Raw cache files (`.dat2`, `.idx`, sprites, models) go in `cache/` (gitignored) or `tools/extracted/` (also gitignored).

## osrsbox Data Location

osrsbox data lives in `data/osrsbox/` (gitignored). The `data/README.md` explains how to populate it. Never commit osrsbox files — they are regenerated by `/import-osrsbox-complete`.

## Wiki Citation in Data Files

Every data value sourced from the OSRS wiki must include a comment with the wiki URL:

```yaml
# data/skills/woodcutting.yaml
trees:
  YEWS:
    level_required: 60      # https://oldschool.runescape.wiki/w/Woodcutting#Trees
    xp: 175.0               # https://oldschool.runescape.wiki/w/Yew_logs
    respawn_ticks: 100      # wiki: ~68 seconds
```

## Prohibited

- No raw binary files in `data/`
- No `.jar`, `.dat2`, `.idx`, `.class` files in `data/`
- No XTEA keys in any `data/` file
- No schema-less JSON files (always add a schema entry)
